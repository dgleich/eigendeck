# Eigendeck Development Log — April 15, 2026

## Overview

Session focused on history panel, hybrid in-memory SQLite, demo export pipeline fixes, and cleanup of legacy JSON tools. Major bug fix: slide doubling on project open.

---

## History Panel (Cmd+Shift+H)

- New `HistoryPanel` component showing timeline of all edits grouped by date
- Click any entry to preview the full presentation state at that timestamp
- Mini slide viewer (scaled 1920x1080) with arrow navigation between slides
- "Restore to this point" button flushes current state first, then overwrites
- Rust: `db_get_history_timestamps()` collects events from elements + slides tables
- Rust: `db_get_state_at(timestamp)` reconstructs full presentation via temporal queries (`valid_from <= ts AND (valid_to IS NULL OR valid_to > ts)`)
- View menu + keyboard shortcut: Cmd+Shift+H

## Orange Glow on Synced Elements

- Selected elements with active `syncId` get orange box-shadow + border (`is-synced` CSS class)
- Visual reminder that edits propagate to other slides

## Hybrid In-Memory/File SQLite

- **Problem**: Couldn't add demos or images before saving a project file
- **Solution**: App starts with `db_open_memory()` — in-memory SQLite DB with full schema
- First save uses SQLite backup API (`db_save_to_file`) to atomically copy to disk, then reopens from file
- All "save project first" gates removed from image/demo add buttons and drag-drop
- Rust: `open_memory_db()` guards against clobbering (no-op if DB already open)

## Demo Asset Loading — SQLite Blob URLs

- Created `src/lib/demoAssets.ts` with shared `useAssetUrl`/`useDemoUrl` hooks
- All asset loading (editor, presenter, separate presenter window) now goes through SQLite `db_get_asset` → blob URL with cache
- Removed `convertFileSrc` from SlideElementRenderer, PresentMode, and presenter.tsx
- Images extracted into `ImageBox` and `PresentImage` components (hooks need components)
- `PropertiesPanel` demo piece scanner reads from SQLite instead of filesystem

## HTML Export Fixes

### Slide centering
- Old: `transform-origin: top left` with manual `paddingTop` centering — broke on narrow windows (black border creep)
- Fix: `position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%) scale(...)`

### Demo-piece export in srcdoc iframes
- **Problem**: `location.hash` and `location.pathname` are empty in srcdoc iframes
- **URLSearchParams patch**: Bootstrap now patches constructor to inject piece/role params when hash is empty
- **BroadcastChannel → postMessage relay**: srcdoc iframes have opaque origins where BroadcastChannel silently fails. Bootstrap replaces BroadcastChannel with a postMessage shim; parent page has a relay script forwarding to all iframes
- **Timing fix**: Relay now broadcasts to ALL iframes (channel keys prevent cross-slide interference). On slide navigation, parent sends `request-state` to re-trigger controller broadcasts

### Demo channel names
- All 12 example demos updated: `location.pathname.split('/').pop()` → hardcoded filenames
- Works in all contexts (blob URL, file URL, srcdoc) without bootstrap patch

## Slide Doubling Bug (Critical Fix)

- **Root cause**: `openSqliteProject()` set `sqliteDbPath` before calling `setPresentation()`, causing the write-through subscriber to see all loaded slides as "new additions" and re-insert them
- **Fix**: Disable subscriber during load (`sqliteDbPath = null`), load data, reset `prevPresentation`, then re-enable
- Restored `magnetic-powers.eigendeck` from git (was doubled from 46 to 92 slides)
- Refreshed all 8 demo HTML assets in the .eigendeck file
